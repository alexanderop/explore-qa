# Brain — core principles

Generic, site-agnostic QA knowledge. Always inlined into the system prompt.
Site-specific knowledge lives under `brain/sites/<site>/` and is inlined only
for the active site.

Inspired by [poteto/brainmaxxing](https://github.com/poteto/brainmaxxing).

## Principles

- [[principles/exploratory-not-checklist]] — Kaner/Bach: learning, design, execution in parallel
- [[principles/absence-burden-of-proof]] — "missing" findings need two independent checks
- [[principles/dom-over-visual]] — DOM counter-check beats visual inspection
